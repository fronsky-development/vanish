## How to Contribute

### Reporting Bugs

1. Check the [existing issues](https://github.com/fronsky-development/vanish/issues) to avoid duplicates.
2. Open a new issue with:
   - Minecraft version and server software (Spigot/Paper)
   - Plugin version (run `/vanish info`)
   - Steps to reproduce the issue
   - Console errors (if any)

### Suggesting Features

Open an issue with the `enhancement` label and describe:
- What the feature should do
- Why it would be useful
- Any implementation ideas (optional)

### Pull Requests

1. Fork the repository.
2. Create a feature branch from `main`.
3. Follow the existing code style and conventions.
4. Test your changes on a local Minecraft server.
5. Submit a pull request with a clear description of the changes.

## Code Style

- Java 17 features are allowed.
- Use the existing project formatting (IntelliJ defaults).
- Add Javadoc to all public methods.
- Keep commits atomic and well-described.
